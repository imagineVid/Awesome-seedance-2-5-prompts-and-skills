## Was ist Seedance 2.5?

**Seedance 2.5** ist die Bezeichnung für die nächste Seedance-Videomodellversion, über die im Juli 2026 berichtet wurde. Der öffentliche Seed-Modellkatalog von ByteDance enthält derzeit weder eine vollständige Modellkarte noch eine stabile API-Kennung oder detaillierte öffentliche Spezifikation für Seedance 2.5. Deshalb trennt dieses Repository bestätigtes Material aus erster Hand von Community-Aussagen und passt seinen Umfang an, sobald ByteDance belastbare Dokumentation veröffentlicht.

Die Prompt-Sammlung konzentriert sich auf überprüfbare Merkmale realer Ergebnisse: Referenzeinsatz, sichtbare Handlung, Kamerapfad, Timing, Dialog- und Tonhinweise, Kontinuität und Schnittabsicht. Zahlen zu Fähigkeiten bleiben ausgeschlossen, bis sie durch eine stabile Quelle aus erster Hand belegt sind.

- **Jeder Referenz eine Aufgabe geben** - Festlegen, welcher Input Identität, Komposition, Bewegung, Ton oder Look steuert
- **Sichtbare Ursache und Wirkung beschreiben** - Jede Aktion mit der erwarteten Bewegung, Reaktion oder Umweltveränderung verbinden
- **Die Szene in Beats gliedern** - Beobachtbare Momente ordnen, statt die Geschichte nur zusammenzufassen
- **Die Tonspur inszenieren** - Dialog, Foley, Atmosphäre und Musik nur dort angeben, wo sie die Szene voranbringen
- **Den Kamerapfad benennen** - Bildausschnitt, Bewegung, Motivabstand und Übergänge zwischen wichtigen Shots festlegen
- **Kontinuität als Regel formulieren** - Identität, Kleidung, Produktgeometrie, Raum und Licht über die Sequenz schützen

**Aktuelle Referenzen:** [Seedance 2.5 release reporting](https://www.theinformation.com/briefings/bytedance-unveils-seedance-2-5-video-model) · [ByteDance Seed model catalog](https://seed.bytedance.com/en/models) · [Available Seedance workflow on ImagineVid](https://imaginevid.io/seedance-2-0)

### Einen Prompt in eine Shot-Vorlage verwandeln

Wiederverwendbare Videoprompts trennen Szenenvariablen von der Regielogik. Ersetze Motiv, Schauplatz, gesprochenen Satz oder Produkt und bewahre dabei den erprobten Kamerapfad, die Beat-Struktur, den Soundplan und die Kontinuitätsregeln.

**Vorlagenmuster:**
```
[DAUER + BILDVERHÄLTNIS]. [MOTIV] führt [SICHTBARE HANDLUNG] in [SCHAUPLATZ] aus. Kamera: [BILDAUSSCHNITT + BEWEGUNG]. Beats: [GETIMTE HANDLUNGEN]. Audio: [DIALOG + FOLEY + ATMOSPHÄRE]. Bewahren: [IDENTITÄT / PRODUKT / LAYOUT]. Vermeiden: [FEHLERMODI].
```

Beginne mit einer Handlung und einer Kameridee. Füge Timing, Audio und Erhaltungsregeln nur hinzu, wenn sie ein sichtbares Produktionsproblem lösen; ändere zwischen den Generierungen jeweils nur eine Variable.
